# Ограничения и допущения

Этот документ описывает ключевые ограничения, принятые в рамках проекта.

## Технические ограничения

- **Язык и окружение**: Только чистый PHP 8.3, без использования пакетного менеджера Composer.
- **Хранение данных**: Данные хранятся исключительно в JSON-файлах. Базы данных (SQL, NoSQL) не используются.
- **Пользователи**: Система рассчитана на одного пользователя (администратора) с доступом по единому паролю.
- **Состояние**: API работает без сохранения состояния между запросами (stateless).
- **Логирование**: Механизмы логирования ошибок и действий отсутствуют.

## Архитектурные запреты

- **Фреймворки**: Запрещено использовать любые PHP-фреймворки (Laravel, Symfony, Slim, Yii и т.д.).
- **ORM**: Запрещено внедрять любые библиотеки для объектно-реляционного отображения (ORM).
- **Конфигурация**: Запрещено изменять конфигурационные файлы (`config.json`, `auth.php`) через API.
- **Прямой доступ к файлам**: Логика API не должна оперировать прямыми путями к файлам; все пути формируются на основе семантических правил и конфигурации.

## Безопасность

- **Аутентификация**: Используется только Basic Auth. Пароль на сервере хранится в виде хеша.
- **Доступ**: API предназначен для использования только из доверенной среды (React Admin Panel). CORS-политики или другие механизмы защиты для публичного доступа не предусмотрены.
- **Типы файлов**: Разрешена загрузка только PDF-файлов и ZIP-архивов. Выполнение кода из загруженных файлов исключено.
- **Валидация**: Все входящие данные должны проходить валидацию по JSON-схемам для предотвращения некорректных данных.
